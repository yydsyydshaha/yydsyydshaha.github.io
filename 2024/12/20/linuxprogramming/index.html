2. 基于文件描述符的文件操作

数据结构 FILE就是文件流，也称为用户态文件缓冲区，所以与FILE类型相关的文件操作（比如fopen，fread等等）称为带缓冲的IO。

在操作文件时，进程会在内存空间的内核区部分里面会维护一个数据结构来管理和文件相关的所有操作，这个数据结构称为打开文件或者是文件对象（filestruct），除此以外，内核区里面还会维护一个索引数组来管理所有的文件对象，该数组的下标就被称为文件描述符(file descriptor)。

2.1 打开创建和关闭文件

open
创建一个文件对象，分配一个文件描述符。

常见的flags的可选项如下：

选项 含义
O_RDONLY 以只读的方式打开
O_WRONLY 以只写的方式打开
O_RDWR 以读写的方式打开
O_CREAT 如果文件不存在，则创建文件
O_EXCL 仅与O_CREAT连用，如果文件已存在，则open失败
O_TRUNC 如果文件存在，将文件的长度截至0
O_APPEND 已追加的方式打开文件，每次调用write时，文件指针自动先移到文件尾，用于多进程写同一个文件的情况。
O_NONBLOCK
O_NDELAY		 对管道、设备文件和socket使用，以非阻塞方式打开文件，无论有无数据读取或等待，都会立即返回进程之中
fopen 底层使用了 open

文本文件和二进制文件
**文本文件**可以用 vim/cat 查看，本质是一串 ASCII 字符序列，如果要查看，直接加载在内存中，则显示的是字符数值。

二进制文件读写所用的数据类型一致即可

而 fscanf 是将文本文件转换为内存中二进制数据

如果文件当中的内容是一串ASCII字符的序列，那么这类文件就是文本文件。对于文本类型的文件，如果使用fread读取到内存当中，那么就应该分配字符串类型的内存区域以存储数据，假若想要根据文件的内容得到相关类型的数据，就需要使用fscanf函数。

如果文件内容是内存数据块按字节为单位直接进行存储，那么该文件就是二进制文件。对于二进制文件，目前只能使用fread和fwrite进行读写操作。对应二进制文件的读写，需要遵循一个基本原则就是：写入是什么类型，读取就使用同样的类型。

2.7 管道

有名管道
:::tips $ mkfifo [管道名字]

使用cat打开管道可以打开管道的读端

$ cat [管道名字]

打开另一个终端，向管道当中输入内容可以实现写入内容

$ echo “string” > [管道名字]

此时读端也会显示内容

:::

（有名）管道文件是用来数据通信的一种文件，它是半双工通信，它在ls -l命令中显示为p，它不能存储数据

有名管道是一种特殊的文件，是一种进程间通信在文件系统的映射(在文件系统存在路径)

通信类型
| 传输方式 | 含义 | | --- | --- | | 全双工 | 双方可以同时向另一方发送数据 | | 半双工 | 某个时刻只能有一方向另一方发送数据，其他时刻的传输方向可以相反 | | 单工 | 永远只能由一方向另一方发送数据 |

管道至少是半双工，一般在使用上，用户会把一条管道当单工使用,如果非要全双工，启用两条管道

open close read write
open 打开管道的一端，`O_RDONLY`读端，`O_WRONLY`写端，open 会引发阻塞，等到对端也被 open 了，才会恢复就绪
管道的read和write

select

:::tips
1.创建监听集合 fd_set rdset;

2.初始化 FD_ZERO

3.把关心的 fd 加入监听 FD_SET

4.调用 select 函数（监听集合，就绪集合）本进程陷入阻塞，内核检查 fd 是否就绪，任一 fd 就绪，select 返回

5.遍历就绪集合 FD_ISSET->read

:::

每次循环都重新设置一下监听
select的实现是通过对设备的轮询来实现的，每次调用FD_ISSET()函数后 ，会把原来待检测的但是仍没就绪的描述字清0了。所以，每次调用select()前要重新调用FD_SET()来设置一下待检测的描述设备。
聊天的关闭
管道写端关闭了，读端处于就绪状态，每次 read 立刻返回 0（相当于发送了一个EOF）

操作系统会将管道的状态设置为可读，可读的状态会影响到select函数，导致select函数不会阻塞，进入死循环。

select的超时机制

select 返回的情况：1.监听的 fd 就绪 2.timeout 超时

每次调用select之前需要重新为timeout赋值，因为调用select会修改timeout里面的内容
